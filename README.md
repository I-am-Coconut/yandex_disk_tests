# Тестирование API Яндекс.Диска


Автоматизированные тесты для проверки основных операций с Яндекс.Диском через REST API.

[![pytest](https://img.shields.io/badge/pytest-7.4.0-green)](https://github.com/pytest-dev/pytest)
[![License](https://img.shields.io/badge/License-MIT-green)

# Описание

Проект содержит набор автоматизированных тестов для проверки корректности работы с API Яндекс.Диска. Тесты покрывают базовые операции:
* получение информации о диске;
* создание и управление папками;
* загрузку и скачивание файлов;
* восстановление файлов из Корзины.

## Документация API

* Официальная документация: https://yandex.ru/dev/disk/api/concepts/about-docpage/
* API для тестирования: https://cloud-api.yandex.net

## Стек технологий

* **Python 3** — язык программирования;
* **Pytest** (https://github.com/pytest-dev/pytest) — фреймворк для написания и запуска тестов;
* **Requests** (https://github.com/psf/requests) — библиотека для выполнения HTTP‑запросов;
* **python-dotenv** — для загрузки переменных окружения из файла `.env`.

## Структура проекта

* ├── .gitignore # Настройки игнорирования файлов Git
* ├── config.py # Конфигурационные параметры и константы
* ├── conftest.py # Фикстуры pytest (подготовка/очистка окружения)
* ├── requirements.txt # Зависимости проекта
* ├── test_token.py # Тест валидности OAuth-токена
* └── test_yandex_disk.py # Основные тесты API Яндекс.Диска


## Установка и настройка

 Предварительные требования

* Python 3.7+;
* установленный Git.

 Клонирование репозитория
* git clone https://github.com/I-am-Coconut/yandex_disk_tests.git
* cd yandex_disk_tests

Установка зависимостей
* pip install -r requirements.txt

Настройка окружения
* Создайте файл .env в корне проекта.

Добавьте в него OAuth‑токен для доступа к API Яндекс.Диска:
* OAUTH_TOKEN=ваш_oauth_токен_здесь

Где взять OAuth‑токен:

* Перейдите в https://oauth.yandex.ru/
* Зарегистрируйте приложение.
* Получите OAuth‑токен с правами доступа к Яндекс.Диску.

# Запуск тестов
* Запуск всех тестов: pytest
* Запуск с подробной информацией: pytest -v
* Запуск тестов с очисткой вывода: pytest -s
* Запуск конкретного теста: pytest test_yandex_disk.py::test_get_disk_info

# Описание тестов
### test_yandex_disk.py
* test_get_disk_info() — проверяет доступ к API и получение базовой информации о диске (общий объём, занятое пространство).

* test_create_and_list_folder() — создаёт подпапку внутри тестовой папки и проверяет её наличие в списке ресурсов.

* test_upload_and_download_file() — тестирует загрузку файла на Яндекс.Диск, его скачивание и проверку содержимого.

* test_restore_from_trash() — проверяет восстановление файла из Корзины после удаления.

### test_token.py
* Проверяет валидность OAuth‑токена: авторизацию в API и получение ответа от сервера.

### Конфигурация (config.py)
Файл содержит:

* базовый URL API Яндекс.Диска;

* заголовки для запросов (авторизация);

* автоматически генерируемые имена тестовой папки и файла (с временной меткой);

* тестовое содержимое файла.

### Фикстуры (conftest.py)
Содержит фикстуру setup_test_folder:

* создаёт тестовую папку перед запуском тестов;

* удаляет тестовую папку после завершения всех тестов.

## Обработка ошибок
### Если тесты не проходят, проверьте:

* корректность OAuth‑токена в файле .env;

* подключение к интернету;

* доступность API Яндекс.Диска;

* права доступа токена;

* логи ошибок в выводе pytest.
